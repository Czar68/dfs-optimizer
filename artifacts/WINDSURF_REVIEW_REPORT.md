## 1. Security

### 1.1 npm vulnerability (minimatch ReDoS) — **CONFIRMED**

- **Finding:** High severity ReDoS in `minimatch` (Jest transitive dependency).
- **Verified:** `npm audit` reports 1 high severity issue; fix available via `npm audit fix`.
- **Action:** Run `npm audit fix`. If breaks tests, pin `minimatch` to a patched version or upgrade Jest.

### 1.2 Exposed API keys / .env — **PARTIALLY ACCURATE**

- **Finding:** “Exposed API keys in .env” and “credentials.json, token.json should be gitignored.”
- **Verified:**
  - `.gitignore` **already includes** `token.json`, `credentials.json`, and `.env` (lines 1, 4, 5).
  - `git check-ignore` confirms `credentials.json` and `token.json` are ignored.
- **Clarification:** The risk is if those files were ever committed before being ignored, or if `.env` is copied elsewhere. They are not “missing from gitignore.”
- **Action:** Ensure `.env` and Google OAuth files are never committed; use env vars in CI (e.g. `TELEGRAM_BOT_TOKEN`, `SGO_API_KEY`). Run `git log -p -- .env credentials.json token.json` to confirm they are not in history; if they are, rotate keys and remove from history.

### 1.3 Sensitive credentials committed — **MISLEADING**

- **Finding:** “credentials.json, token.json should be gitignored.”
- **Verified:** Both are already in `.gitignore`. The review is redundant unless it detected these files in the working tree or in history.
- **Action:** If the files exist on disk, keep them local-only and never `git add` them. No change needed to `.gitignore`.

## 2. Git repository

### 2.1 Corrupted repo — **CONFIRMED**

- **Finding:** “Corrupted Git repository - required git init to fix.”
- **Verified:** `git status` returns `fatal: bad object HEAD`, which indicates a broken repo (e.g. missing/corrupt HEAD or refs).
- **Action:** From repo root:  
  - Back up uncommitted work.  
  - Try `git fsck` to see which objects are broken.  
  - If refs/HEAD or the current branch ref is bad, restore from backup or re-clone and re-apply changes.  
  - Avoid running `git init` in an existing repo unless you intend to reinitialize and lose history; prefer repairing refs/objects.

### 2.2 Incomplete .gitignore — **CONFIRMED**

- **Finding:** Missing patterns such as `__pycache__/`, `*.pyc`, `.env.local`.
- **Verified:** `.gitignore` does not contain `__pycache__/`, `*.pyc`, or `.env.local`.
- **Action:** Add to `.gitignore`:
  - `__pycache__/`
  - `*.pyc`
  - `.env.local`
  - Optionally `*.pyo`, `.env.*.local`, and IDE/project dirs if desired.
